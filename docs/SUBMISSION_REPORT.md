## 1. 한 줄 요약

"주가가 앞으로 어떻게 움직일지 300만 번 시뮬레이션해서 확률 분포로 보여주는 프로그램을 만들었는데, 잘 맞는 종목도 있고 전혀 맞지 않는 종목도 있었다."

---

## 2. 왜 만들었나

주식을 사기 전에 "이 종목이 1년 뒤 오를 가능성이 얼마나 되나?"를 수치로 보고 싶었다. 단순히 "오를 것 같다"는 감이 아니라, 과거 데이터를 바탕으로 계산한 확률과 손실 범위를 한눈에 보여주는 도구를 직접 구현했다.

---

## 3. 어떻게 작동하나

### 3.1 기본 아이디어 — "동전 던지기 300만 번"

주가는 매일 조금씩 오르거나 내린다. 오르고 내리는 폭은 그 종목의 역사적 변동성(얼마나 크게 흔들렸는가)을 기준으로 정해진다. 이 규칙대로 미래 1년을 무작위로 그려보는 시뮬레이션을 300만 번 반복하면, "이런 가격이 될 가능성은 몇 퍼센트"라는 분포를 얻을 수 있다.

수식으로는 GBM(기하 브라운 운동)이라는 확률 모델을 사용한다.

$$
S_{t+1} = S_t \times e^{\left(\mu - \frac{\sigma^2}{2}\right)\Delta t \;+\; \sigma\sqrt{\Delta t}\,Z}
$$

- $S_t$: 현재 주가
- $\mu$: 연간 기대 수익률 (얼마나 오를지)
- $\sigma$: 연간 변동성 (얼마나 흔들리는지)
- $Z$: 매번 새로 뽑는 무작위 숫자

$\mu - \sigma^2/2$ 항이 중요하다. 이 항이 없으면 변동성이 클수록 자동으로 낙관적인 경로가 만들어지는 수학적 오류가 생긴다. 이를 보정해주는 항이다.

### 3.2 파라미터를 어떻게 정하나

야후 파이낸스에서 최근 2년치 실제 주가를 내려받아 다음을 계산한다.

- $\sigma$ (변동성). 일별 수익률의 표준편차에서 계산. 최근 변동성이 더 중요하므로 EWMA 방식으로 최근 데이터에 가중치를 더 준다.
- $\mu$ (기대 수익률). 일별 수익률 평균에서 계산. 단, 그대로 쓰면 과거 급등이 미래에도 계속될 것처럼 과장되므로, "시장 평균 연 8%"와 50:50으로 섞어 보수적으로 조정한다.

### 3.3 속도 문제 — 왜 C++을 쓰나

300만 번 시뮬레이션을 Python만으로 돌리면 약 0.26초가 걸린다. 작아 보이지만 사용자가 버튼을 누를 때마다 체감 딜레이가 생긴다. C++로 핵심 계산 부분을 작성하고 Python에서 호출하는 방식으로 바꾸니 0.038초로 줄었다(6.8배 빠름).

| 경로 수 | C++ | Python(NumPy) | 차이 |
|---:|---:|---:|---:|
| 10만 개 | 0.0013초 | 0.0016초 | 비슷 |
| 100만 개 | 0.0058초 | 0.022초 | **3.8배** |
| 1000만 개 | 0.038초 | 0.260초 | **6.8배** |

단, 10만 개 이하에서는 C++ 초기화 비용 때문에 오히려 Python이 빠르다.

### 3.4 산출 지표

| 지표 | 쉬운 설명 |
|:---|:---|
| 상승 확률 | 300만 경로 중 1년 뒤 현재가보다 오른 비율 |
| VaR(95%) | 최악의 5% 상황에서 잃는 금액 |
| CVaR(95%) | 최악의 5% 상황의 평균 손실 (VaR보다 더 보수적) |
| Kelly 비중 | 기댓값 기준 최적 투자 비율 |
| 팬 차트 | 시간에 따른 가격 분포 띠 — 넓을수록 불확실성 큼 |

| 종목 | 방향 정확도 | 중앙값 오차 | 90% 구간 범위 |

## 5. 실패 경험 — 만들면서 틀린 것들

### 5.1 드리프트를 그대로 쓰면 말이 안 되는 예측이 나온다

처음에는 과거 수익률 평균을 그대로 $\mu$로 사용했다. NVDA는 2023~2024년 AI 붐으로 연간 +200%가 넘었기 때문에 이 기간 데이터를 쓰면 $\mu$가 200%로 설정되어 "1년 뒤 가격 = 지금의 3배"라는 예측이 나왔다.

**해결.** 추정된 $\mu$와 시장 평균 8%를 50:50으로 섞었다. 또 $\mu$의 상한을 20%로 제한했다.

**부작용.** TSLA처럼 실제로 고성장하는 주식에서도 $\mu$가 20%로 잘린다. 그런데 변동성이 60%이면 수식 속 $\sigma^2/2 = 0.18$이 $\mu = 0.20$을 거의 다 상쇄해서 실질 드리프트가 약 2%밖에 안 남는다. 결국 상승/하락이 50:50에 가까워진다.

**테슬라 상승 확률이 54%인 이유가 이것이다.** 변동성이 너무 크면 수학 구조 자체가 상승 확률을 50%로 끌어내린다. 주관적으로 전망이 좋아 보여도 모델은 "모른다"고 말하는 셈이다.

### 5.2 커버리지 100%는 잘 맞는 게 아니라 오히려 문제다

백테스트에서 "실제 가격이 예측 구간 안에 들어온 비율(커버리지)"을 재봤더니 TSLA, AAPL 모두 100%가 나왔다. 처음엔 좋은 결과처럼 보였지만, 이건 오히려 **모델이 쓸모없다는 신호**다.

**비유.** 날씨 예보에서 "내일 기온은 –10°C~50°C 사이일 것이다"라고 하면 항상 맞는다. 하지만 이 예보는 아무 정보도 주지 않는다. 좋은 예보는 좁은 범위로 맞추는 것이다.

**실제 TSLA 90% 예측 구간.** $162~$1,176. 현재가 $428 기준으로 –62%~+175%. 이 범위 안에 들어오지 않는 가격이 거의 없다. 구간이 지나치게 넓은 이유는 변동성(σ=60%)이 너무 크기 때문이다.

**이 문제의 원인.** GBM 모델은 변동성이 클수록 예측 구간이 기하급수적으로 넓어지도록 설계되어 있다. 변동성 자체를 더 정교하게 예측하거나, 미래의 변동성이 줄어드는 패턴을 반영하지 않으면 구조적으로 해결되지 않는다.

**미해결 과제다.** 이상적인 커버리지는 90%여야 한다. 100%는 "항상 맞추는 쓸모없는 예보"와 같다.

### 5.3 NVDA 급등을 전혀 예측 못했다

NVDA는 AI 붐으로 2023년에 주가가 3배 올랐다. 이 시기 데이터로 1년 뒤를 예측해도 모델은 +14%라고 했고, 실제는 +150%이었다. 오차가 77%에 달한다.

**이유.** GBM은 "앞으로도 과거와 비슷하게 움직일 것"이라고 가정한다. AI 반도체 수요 폭발처럼 완전히 새로운 이유로 주가가 급등하는 상황은 과거 데이터에 존재하지 않으므로 반영할 방법이 없다. 이 한계는 파라미터 조정으로 해결되지 않는다.

### 5.4 차트가 마우스를 올리면 밀려나는 현상

Streamlit에서 Plotly 차트를 그렸더니 마우스를 올릴 때 차트가 움직이거나 확대되는 현상이 있었다. 사용자가 고정된 이미지처럼 보면서 값만 hover로 확인하길 원했다.

처음엔 matplotlib으로 교체해 PNG로 고정했는데, 이번엔 hover 툴팁 자체가 사라져 월별 금액 확인이 불가능해졌다. 결국 Plotly의 `fixedrange=True` 옵션으로 줌·패닝을 막으면서 hover는 살리는 방식으로 해결했다.

### 5.5 큰 숫자가 읽기 불편했다

₩1000000000처럼 쉼표 없이 표시되면 자릿수를 세기 어렵다. Streamlit의 숫자 입력 컴포넌트가 `format=",.0f"` 옵션을 제대로 지원하지 않아, 입력 필드 위에 포맷된 금액(`₩1,000,000,000`)을 HTML로 따로 보여주는 방식으로 처리했다.

## 7. 잘 됐던 것

AAPL처럼 변동성이 낮고(σ≈28%) 꾸준히 성장한 종목에서는 5년 백테스트 기준 방향 정확도 100%, 중앙값 오차 9.4%를 달성했다. GBM 모델의 가정("과거와 비슷하게 움직인다")이 실제에 가까운 종목에서는 꽤 잘 작동한다.

C++ 엔진도 성공적이다. 1000만 경로를 0.038초에 처리하면서 대시보드 버튼을 누를 때 체감 대기 시간이 없다.

## 8. 실행 방법

### 사전 조건

- Python 3.11 이상 ([python.org](https://python.org) 에서 설치)
- Git ([git-scm.com](https://git-scm.com) 에서 설치)

C++ 엔진은 선택 사항이다. 없으면 자동으로 Python(NumPy) 모드로 동작한다. 속도는 조금 느리지만 결과는 동일하다.

### 1단계 — 코드 받기

```bash
git clone https://github.com/leejunhyeop2001/stellar-quant.git
cd stellar-quant
```

### 2단계 — 패키지 설치

```bash
pip install -r requirements.txt
```

### 3단계 — 웹 대시보드 실행

```bash
streamlit run python/dashboard.py
```

실행하면 터미널에 `http://localhost:8501` 주소가 출력된다. 브라우저에서 이 주소로 접속하면 된다.

### (선택) C++ 엔진 빌드 — 속도 개선용

C++ 엔진을 빌드하면 300만 경로 계산이 0.26초 → 0.04초로 빨라진다. 빌드하려면 추가 도구가 필요하다.

- Windows: [Visual Studio 2022](https://visualstudio.microsoft.com/) (C++ 워크로드 포함) + cmake
- macOS: `xcode-select --install` + cmake
- Linux: `sudo apt install build-essential cmake`

```bash
cmake -S . -B build
cmake --build build --config Release   # Windows
# cmake --build build                  # macOS / Linux
```

빌드 후 streamlit을 재시작하면 자동으로 C++ 엔진을 사용한다.

## 9. 결론

이 프로젝트를 통해 "수치로 만들어보면 직관과 다른 결과가 나온다"는 것을 직접 경험했다. 테슬라처럼 주관적으로 전망이 좋아 보이는 종목이어도 변동성이 너무 크면 모델은 54%라는 애매한 확률을 출력한다. 이것이 모델의 결함이기도 하지만, 반대로 "과거 데이터만 봐서는 미래를 확신할 수 없다"는 정직한 신호이기도 하다.

가장 큰 미해결 문제는 **예측 구간이 지나치게 넓다**는 점이다. 구간 안에 항상(100%) 들어오는 예측은 맞추는 것이 아니라 아무 말도 하지 않는 것과 같다. 변동성이 높은 종목일수록 더 좁고 유용한 예측 구간을 만드는 것이 다음 단계 과제다.

**이 프로그램은 투자 권유가 아니다.** 과거 데이터를 바탕으로 한 통계 실험이며, 실제 투자 결정에는 여기서 다루지 않은 수많은 요소가 필요하다.
