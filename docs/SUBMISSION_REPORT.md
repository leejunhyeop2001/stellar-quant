## 1. 요약 (Abstract)

본 프로젝트는 **기하 브라운 운동(GBM)** 과 **Merton 점프 확산**을 가정한 주가 모델 위에서 **몬테카를로 시뮬레이션**을 수행하여, 미래 시점 주가의 분포·분위수·상승 확률·VaR(95%) 등을 추정하는 하이브리드 응용 프로그램이다. 계산 집약 구간은 C++23 멀티스레드로 구현하고, 데이터 수집·통계·시각화·UI는 Python으로 구성하였다. pybind11을 통해 C++ 모듈을 Python에서 호출하며, 시뮬레이션 구간에서는 GIL 해제로 병목을 줄였다. 사용자는 **Streamlit 웹 대시보드** 또는 **CLI** 로 동일한 엔진 코어 결과를 확인할 수 있다.

**최종 보고서에서 추가된 내용.** 1·2차 보고서 이후 수행한 실제 실험 결과, 백테스트 정량 평가, 실패 경험과 개선 과정을 포함하였다. 실험 결과 모델은 AAPL처럼 변동성이 낮고 꾸준히 성장한 종목에서는 방향 정확도 100%, 중앙값 오차 9.4%를 달성한 반면, TSLA처럼 변동성이 극단적으로 높은 종목에서는 방향 정확도가 50%(동전 던지기 수준)에 그쳤다. 이 간극이 본 프로젝트가 극복하지 못한 핵심 한계이다.

## 2. 연구·개발 목적 및 배경

### 2.1 목적

- 실제 시장 데이터(Yahoo Finance)로부터 초기 가격 $S_0$, 연율 드리프트 $\mu$, 연율 변동성 $\sigma$를 추정하고, 이를 GBM 가정 하에 대규모 난수 시뮬레이션으로 미래 가격 분포를 근사한다.
- 단순 스크립트 수준이 아니라, 고성능 코어(C++)와 분석·표현 계층(Python)을 분리한 구조로 확장성과 실행 속도를 동시에 확보한다.
- 금융 실무에서 자주 쓰는 지표(분위수, 상승 확률, VaR, CVaR, Kelly, Sortino)를 비모수적(시뮬레이션 샘플 기반)으로 산출하여 시각적으로 전달한다.

### 2.2 배경

주가 모델링에서 GBM은 로그수익률의 정규성을 가정하는 대표적인 확률 모델이다. 해석해를 이용하면 종료 시점 $T$의 주가는 로그정규 분포를 따르며, 다단계 경로(Fan Chart)는 정확 이산화(exact step)로 시간축을 따라 시뮬레이션할 수 있다. Python 순환만으로는 대규모 경로(예: 300만 개) 처리에 한계가 있으므로, C++ 병렬 RNG 및 지수 갱신으로 성능을 확보한다.

## 3. 이론적 기반

### 3.1 GBM 확률미분방정식 (SDE)

$$
dS_t = \mu S_t\,dt + \sigma S_t\,dW_t
$$

- $\mu$: 연율화 기대수익률(drift)
- $\sigma$: 연율화 변동성(volatility)
- $dW_t$: 위너 과정 증분

### 3.2 이산화 — Itô 보조정리 적용

$$
S_{t+\Delta t} = S_t \exp\!\left[\left(\mu - \tfrac{1}{2}\sigma^2\right)\Delta t + \sigma\sqrt{\Delta t}\,Z_t\right], \quad Z_t \sim \mathcal{N}(0,1)
$$

$\frac{1}{2}\sigma^2$ 항을 빼는 이유. 로그정규 분포에서 기댓값이 $e^{\mu t}$가 되려면 지수 안의 드리프트가 $\mu - \frac{1}{2}\sigma^2$이어야 한다. 이 **Itô 보정**을 생략하면 변동성이 클수록 실제보다 낙관적인 경로가 체계적으로 생성된다.

### 3.3 파라미터 추정 파이프라인

```
일별 로그수익률 r_t = ln(P_t / P_{t-1})
       ↓
σ̂_역사 = std(r_t) × √252
σ̂_EWMA: σ²_{t} = 0.94·σ²_{t-1} + 0.06·r²_t  (최근 변동성 클러스터 반영)
σ_최종 = max(σ̂_역사, 0.7·σ̂_역사 + 0.3·σ̂_EWMA)
       ↓
μ̂_연율 = mean(r_t) × 252
μ_수축 = 0.5·μ̂ + 0.5·0.08  (베이지안 시장 prior 8%와 혼합)
       ↓
최종 클램프: μ ≤ 20%, σ ≥ 15%
```

### 3.4 Merton 점프 확산

일별 수익률 중 $|r_t - \mu_d| > 3.5\,\sigma_d$ 인 이상치를 점프로 탐지한다. 점프 강도 $\lambda$ (연간 점프 횟수 추정), 평균 점프 크기 $\mu_j$, 점프 표준편차 $\sigma_j$를 추정하여 대시보드에서 선택적으로 적용할 수 있다.

### 3.5 리스크 지표

| 지표 | 정의 | 비고 |
|:---|:---|:---|
| 상승 확률 | $P(\log S_T > \log S_0)$ 의 시뮬레이션 비율 | |
| VaR(95%) | $S_0 - Q_{0.05}(S_T)$ | 하위 5% 분위 손실 |
| CVaR(95%) | $E[\max(S_0-S_T, 0) \mid S_T \leq Q_{0.05}]$ | 꼬리 손실 기댓값 |
| Kelly 비중 | $f^* = \text{clip}\!\left(\frac{\mu - r_f}{\sigma^2},\,0,\,1\right)$, $r_f=3\%$ | 최적 레버리지 |
| Sortino | $\frac{E[\log r] - \log(1+r_f)}{\sigma_\text{하방}}$ | 하방 분산만 사용 |
| 초과첨도 | $\kappa = E[z^4] - 3$ | 꼬리 두께 |
| 위험 점수 | $\frac{VaR/S_0}{\sigma\sqrt{T}} \times (1 + 0.1 \cdot \max(0,\kappa))$ | 정규화 위험 지수 |

## 7. 실패 경험 및 개선 과정

### 7.1 실패 1 — 순수 Python 구현의 속도 한계

**1차 시도.** NumPy 배열 연산으로 100만 경로를 생성했을 때 약 0.26초 소요. 사용자가 시뮬레이션 버튼을 누를 때마다 UI가 멈추는 현상 발생.

**문제 진단.** GIL 때문에 Python 멀티스레딩이 효과 없었고, `multiprocessing`은 프로세스 fork 오버헤드로 오히려 느렸다.

**해결.** `std::thread` + `std::mt19937` 스레드별 독립 난수 생성기를 C++23으로 구현. 10M 경로에서 0.038초(6.78× 개선).

**미해결.** 소규모(10만 경로 미만)에서는 C++ 초기화 오버헤드 때문에 NumPy가 더 빠르다. Fan Chart용 경로(3,000개)는 여전히 Python이 더 유리하다.

---

### 7.2 실패 2 — 드리프트 과대 추정으로 인한 비현실적 예측

**1차 시도.** 역사적 로그수익률 평균 그대로 μ를 추정했더니 TSLA μ̂ ≈ 40%, NVDA μ̂ ≈ 80%(2023~2024 AI 붐 반영)가 나왔고, 중앙값 예측이 현재가의 2~3배로 나왔다.

**문제 진단.** 역사적 평균 드리프트는 표본 잡음이 매우 크다. 252 거래일의 평균은 표준오차가 $\sigma/\sqrt{252} \approx 40\%/15.9 \approx 2.5\%$이므로, 연간 드리프트 추정치의 신뢰구간이 ±5~10%포인트에 달한다.

**해결 1 — 베이지안 수축.** 추정 μ와 시장 평균 prior 8%를 50:50 혼합.

$$
\mu_\text{수축} = 0.5 \cdot \hat\mu + 0.5 \cdot 0.08
$$

**해결 2 — 클램프.** 수축 후에도 μ > 20%이면 20%로 상한 제한.

**부작용(한계).** TSLA처럼 실제로 시장 평균을 크게 초과하는 종목에서 이 수축이 과도하게 보수적인 예측을 만든다. 원래 μ̂=40%인 경우 수축 후 24%, 클램프 후 20%가 되는데, Itô 보정에서 $\sigma^2/2 = 0.60^2/2 = 0.18$이 빠지면 실질 드리프트는 $0.20 - 0.18 = 0.02$, 즉 사실상 0에 수렴한다.

**이것이 TSLA 상승 확률 53.6%의 구조적 원인이다.** 변동성이 클수록 Itô 보정이 드리프트를 더 많이 상쇄하고, 결과적으로 상승 확률이 50%에 가까워진다. 사용자가 주관적으로 강세 전망을 가진 종목이어도 모델은 50% 언저리를 출력한다. 이를 "잘못된" 것으로 볼 수도 있지만, 반대로 "과거 데이터만으로는 미래를 확신할 수 없다"는 정직한 신호로 해석할 수도 있다.

### 7.3 실패 3 — NVDA 급등 미포착

**현상.** NVDA는 2023년부터 AI 수요 폭발로 주가가 1년 만에 2~3배 상승했다. 백테스트에서 방향 정확도는 100%(상승 방향은 맞춤)이지만 중앙값 오차는 77.2%에 달한다. 모델은 +14%를 예측했지만 실제는 +150%이었다.

**근본 원인.** GBM은 파라미터가 시간에 따라 일정하다고 가정한다. 새로운 기술 패러다임(AI 반도체 수요 폭증)처럼 **구조적 레짐 변화(regime shift)**는 과거 σ·μ로 포착되지 않는다.

**시도한 대응.** Merton 점프 모델 추가로 급등 이벤트 일부를 λ(점프 강도)로 모델링했으나, 1년 이상 지속되는 상승 추세는 점프가 아니라 레짐 변화이므로 여전히 미포착이다.

**결론.** GBM 계열 모델은 "평균 회귀적" 주식(대형주, 안정적 성장주)에서는 유효하지만, 패러다임 전환 중인 종목에서는 구조적으로 낙관치를 저평가한다. 이 한계는 파라미터 조정만으로 해결되지 않는다.

---

### 7.4 실패 4 — 초기 Fan Chart의 UX 문제

**1차 시도.** Plotly 기본 설정으로 Fan Chart를 렌더링했더니 마우스를 올릴 때 차트가 패닝(이동)되어 보고서처럼 고정된 이미지를 원하는 사용자가 불편했다.

**2차 시도.** matplotlib으로 기존 Plotly 차트를 교체하여 PNG로 base64 인코딩해 렌더링했다. 이동은 없어졌지만 hover 툴팁도 사라져 월별 가격 확인이 불가능해졌다.

**최종 해결.** Plotly를 유지하되 `xaxis_fixedrange=True`, `yaxis_fixedrange=True`, `dragmode=False`로 축 고정, `hovermode='x unified'`로 hover만 활성화. 패닝·줌 없이 툴팁만 동작.

### 7.5 실패 5 — 투자 금액 입력 UX

**문제.** `st.number_input`에서 `format=",.0f"`를 쓰면 쉼표 구분이 화면에 표시되지 않고 원시 숫자만 보인다. 한국 사용자는 큰 금액(1억 원 등)을 숫자만으로 보면 자릿수를 읽기 어렵다.

**해결.** 입력 필드 위에 HTML로 포맷된 금액(`₩1,000,000,000`)을 별도 표시하고, 입력 자체는 `format="%.0f"`로 단순화. 시각적 피드백과 기능적 입력을 분리.

## 8. 현재 시스템의 한계 종합

### 8.1 구조적 한계 (해결 불가)

| 한계 | 설명 |
|:---|:---|
| GBM 가정 | 로그수익률 정규성 가정. 실제 시장의 두꺼운 꼬리(fat tail), 변동성 클러스터링, 레버리지 효과를 포착 못함 |
| 일정 파라미터 | μ·σ가 시간에 따라 변하지 않는다고 가정. 레짐 변화(NVDA 사례) 미포착 |
| Itô 상쇄 | 고변동성 종목에서 μ_cap(20%) - σ²/2(≈18%)로 실질 드리프트가 0에 수렴 → 상승 확률 구조적 50% 수렴 |
| 과거 의존성 | μ·σ 모두 과거 2년 데이터 기반. 미래 전망(신제품, 정책 변화 등) 반영 불가 |

### 8.2 수치로 드러난 한계

- **TSLA 방향 정확도 50%.** 통계적으로 의미 없는 예측 수준
- **NVDA 중앙값 오차 77%.** 급등 종목에서는 절대값 자체가 크게 틀림
- **커버리지 100%의 역설.** TSLA 90% 구간이 $148~$1,404이므로 실제 가격이 구간 밖으로 나가기 불가능에 가까움. 넓은 구간은 정확성이 아니라 무지(uncertainty)의 표현
- **모든 고변동성 종목에서 μ가 상한에 클램프.** TSLA, NVDA, 삼성전자 모두 μ=20% 상한에 걸림 → 종목 고유의 성장성이 구별되지 않음

### 8.3 보완된 부분

| 문제 | 보완 방법 |
|:---|:---|
| 드리프트 과대추정 | Bayesian shrinkage + clamp |
| 최근 변동성 반영 | EWMA σ 블렌딩 |
| 급등락 이벤트 | Merton 점프 확산 (선택 적용) |
| 분포 꼬리 두께 | 초과첨도 기반 위험 점수 조정 |
| 분산 감소 | Antithetic variates (±Z 쌍) |
| 표본 잡음 | 대규모 경로(300만) |

## 10. 결론

본 프로젝트는 GBM·Merton 점프 확산 기반 몬테카를로 시뮬레이션을 C++/Python 하이브리드로 구현하여, 대규모 샘플 처리(10M 경로, 0.038초)와 풍부한 시각화·UI를 달성하였다. 다만 실험 결과 모델의 유효성은 종목 특성에 크게 의존한다.

**모델이 유효한 구간.** AAPL처럼 변동성이 중간 수준(σ≈28%)이고 꾸준히 성장한 종목에서는 방향 정확도 100%, 중앙값 오차 9.4%를 달성한다. GBM의 가정(일정 파라미터, 로그정규 분포)이 실제에 가까운 조건이다.

**모델의 한계.** TSLA(σ≈60%)처럼 고변동성 종목은 Itô 보정이 드리프트 상한을 상쇄하여 상승 확률이 구조적으로 50%에 수렴한다. NVDA처럼 레짐 변화가 발생한 종목은 중앙값 오차가 77%에 달한다. 이 두 한계는 파라미터 조정이나 경로 수 증가로 해결되지 않으며, 확률적 변동성(Heston 모델), 레짐 전환 모델(HMM), 뉴스·펀더멘털 정보를 통합하는 방향으로 모델을 고도화해야 극복할 수 있다.

**투자 판단에 대한 경고.** 이 시스템이 출력하는 수치는 과거 데이터 기반 통계 추정값이다. "주관적으로 전망이 좋다"는 판단은 모델이 포착하지 못하는 정보이며, 본 소프트웨어는 교육·연구용 시뮬레이션 도구이지 투자 권유가 아니다.
